\chapter{CLI Reference}\label{app:cli}

\begin{verbatim}
runc spec
    # Emits config.json with an empty process.capabilities object
    # (new default; was the upstream three-cap set).

runc run [--default-capabilities] [--default-capabilities-docker]
         [--security-scan]
         [--scan-seccomp-hook PATH]
         [--scan-capable      PATH]
         [--scan-bpftool      PATH]
         <container-id>

  --default-capabilities
       Restore the upstream runc default cap set (3 caps).
       Mutually exclusive with --default-capabilities-docker.

  --default-capabilities-docker
       Restore the historical Docker default cap set (14 caps).
       Mutually exclusive with --default-capabilities.

  --security-scan
       Record the workload via eBPF (oci-seccomp-bpf-hook for
       syscalls, capable-bpfcc --cgroupmap for capabilities,
       AppArmor in complain mode for filesystem accesses).
       Writes everything under <bundle>/generated/.

  --scan-seccomp-hook PATH
       Override the auto-detected oci-seccomp-bpf-hook binary.
       Mostly useful for the contrib stub in CI smoke runs.

  --scan-capable PATH
       Override the auto-detected capable-bpfcc binary.

  --scan-bpftool PATH
       Override the auto-detected bpftool binary.
\end{verbatim}

\chapter{Generated Bundle Layout}\label{app:bundle}

After a successful \texttt{runc run -{}-security-scan}, the bundle contains:

\begin{verbatim}
<bundle>/
  config.json                              # narrowed in place by finalise
  rootfs/                                  # untouched
  generated/
    seccomp.json                           # allow-list seccomp profile
    capable-bpfcc.log                      # cgroup-wide capability trace
    capabilities-from-proc-status.txt      # poststart /proc/<init>/status
    apparmor.profile                       # complain-mode profile
    apparmor-load.log                      # apparmor_parser stdout/stderr
    apparmor-README.txt                    # how to refine via aa-logprof
    spec.original.json                     # config.json before narrowing
    .runc_cap_trace.pid                    # tracer PID (cleaned at stop)
\end{verbatim}

No file under \texttt{generated/} is executable; the bundle gains no script.

\chapter{Reproducing the Matrix Locally}\label{app:repro}

\begin{verbatim}
# 1. Build the runtime fork.
cd runc
make

# 2. Run the smoke pack against the contrib stubs.
sudo make localintegration TESTPATH=/security_scan.bats

# 3. Drive a real workload (requires bpfcc-tools, oci-seccomp-bpf-hook,
#    apparmor-utils, cgroup-v2 host).
sudo ./runc run --security-scan <container-id>
ls bundle/generated/

# 4. Re-run the same workload in enforcement mode using the generated
#    profiles (pin them in config.json or via spec overrides).
sudo ./runc run <container-id>
\end{verbatim}

For the full laboratory workflow, see the \texttt{runc-hardened-test} repository
(\texttt{scripts/setup.sh}, \texttt{scan.sh}, \texttt{apply.sh}, \texttt{run-apparmor-public.sh}).
